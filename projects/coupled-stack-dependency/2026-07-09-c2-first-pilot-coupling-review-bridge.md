## Bridge Question

The C2 evidence-packet checklist already says the immediate review question is
whether existing artifacts show isolated-module failures. The RQ5 bundle now
organizes the first-pilot source-trace prep, and the second-ring stop stack
organizes the protected decisions that remain blocked before live use.

The bridge question is:

```text
Where does the current first-pilot package become misleading if a reviewer
reads one layer alone instead of reading the coupled stack?
```

This is narrower than "is C2 true?" It only identifies where the current
scaffolds make coupling pressure visible enough for later governed review.

## What The First-Pilot Stack Adds To C2

The earlier C2 artifacts were mostly structural: they mapped layers, fixture
coverage, and review-packet organization. The current first-pilot stack adds a
more concrete stress point:

```text
A synthetic contribution packet can look locally coherent while still being
blocked by source-trace gaps, live-record selection, governance authority,
rights meaning, capture response, AI authority, reward meaning, and transfer
language.
```

That is useful for C2 because isolated modules can each appear reasonable:

- the eligibility layer can say an attempt is allowed;
- the taxonomy layer can classify the work;
- the value layer can name useful substance;
- the source-trace layer can list missing fields;
- the log layer can describe possible record states;
- the rights layer can preserve contestability language;
- the reward layer can preserve no-current-reward language;
- and the governance layer can preserve authority stops.

The coupled-stack question is whether these meanings stay separated and
traceable when read together. The current scaffolds make that question visible.
They do not answer it for real contributors.
