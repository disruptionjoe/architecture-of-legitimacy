Status: review-prep scaffold, not adopted governance.

Purpose: prepare a bounded review object for T7 without changing founder powers, transition milestones, governance policy, claim status, reward meaning, or real contribution records.

This file does not specify the project's actual founder powers, limits, transition triggers, rollback rules, emergency powers, participant rights, reviewer authority, or appeal rules. It identifies what a later governance review must decide before T7 can be treated as tested.

## T7 Review Question

Can the repo specify founder powers, limits, and transition triggers clearly enough that founder-led governance becomes visible, bounded, contestable, and non-theatrical?

For this prep scaffold, "specify" means "prepare the decision slots and evidence requirements for later review." It does not mean this run adopts the answers.

## Evidence Requirements For A Later T7 Pass

A later T7 review should be able to inspect a compact packet with:

- a list of named founder powers and whether each is editorial, contribution-review, claim-status, governance, public-posture, or emergency authority;
- a minimum rationale requirement for each power class;
- a contest path for adverse contribution or governance-relevant decisions;
- a phase-transition evidence table that separates "ready to pilot" from "ready to transfer power";
- a record-retention rule for contributors who exit or lose a contest;
- explicit non-adoption language for any power that is only being explored;
- and at least one capture-risk note for each transfer candidate.

The packet should avoid importing governance machinery from another repo unless the local evidence shows why this repo needs it.

## Review Prompts

Use these prompts when a future run or human review turns this prep scaffold into a real T7 test:

1. Which founder powers already exist in practice but are not named with enough precision?
2. Which powers can be constrained immediately without slowing useful research?
3. Which powers cannot be transferred until real contribution records exist?
4. Which transition milestones are observable rather than symbolic?
5. Which appeal or contest paths can be tested before any reward mechanism exists?
6. Which emergency powers are genuinely needed, and how would they expire?
7. What evidence would show that founder-led governance is losing legitimacy despite formal compliance?
8. What evidence would show that early transfer increased capture risk?

## Pass Criteria For A Later T7 Review

T7 can move toward a stronger result when:

- founder powers are named at decision-level granularity, not only as a broad founder-led mode;
- each power has at least one visible rationale or constraint mechanism;
- contributors can tell which decisions can be contested and how;
- phase transitions use observable evidence rather than vibes or time elapsed;
- emergency or rollback authority is bounded and reviewable;
- transfer candidates name capture risks before transfer;
- and the project can explain why it remains founder-led without presenting that state as already decentralized.

This scaffold does not mark T7 passed. It gives the repo a bounded object for future governance review.
