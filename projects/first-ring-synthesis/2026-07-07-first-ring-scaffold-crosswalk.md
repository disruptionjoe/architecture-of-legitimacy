# First-Ring Scaffold Crosswalk

Status: synthesis scaffold, not adopted contribution, review, governance, reward, rights, or workflow policy.

This crosswalk synthesizes existing first-ring scaffolds so a later reviewer can
see what the repo has already prepared, what remains synthetic or unresolved,
and which decisions remain gated. It does not change active contribution
instructions, issue templates, review policy, reviewer authority, response-time
promises, appeal rights, contribution-log schema, governance, reward meaning,
claim status, test pass/fail state, public posture, or any real contribution
record.

## Why This Exists

The repo now has many useful first-ring scaffolds. More isolated scaffolding is
less useful than one map that shows how the current pieces relate.

The first-ring question is no longer only:

```text
Does each local scaffold exist?
```

It is now:

```text
Can a future first-pilot review preserve eligibility, class, value, rationale,
workflow state, log eligibility, useful residue, and protected boundaries as one
trace without private founder context?
```

This file is only a review-prep synthesis. It is not an invitation, policy, or
workflow launch.

## Shared Pattern Across The Scaffolds

The first-ring scaffolds converge on one discipline:

```text
Preserve useful value and visible reasoning without converting review-prep into
authority, reward, rights, policy, public claim strength, or live contribution
state.
```

The repeated local moves are:

1. Name the submitted or synthetic object before interpreting it.
2. Assign one primary contribution class, with secondary effects only when they
   explain a real ambiguity.
3. Tie evidence expectations to the class and target.
4. Separate useful substance from protected state movement.
5. Record the decisive rationale and what was explicitly not decided.
6. Preserve loss notes or useful residue without making a promise.
7. Keep synthetic examples outside live `CONTRIB-*` records.
8. Name contestable points without adopting appeal rights, timing commitments,
   reviewer-selection rules, or governance process.

## First-Pilot Trace Burden

A later first-pilot packet would need to carry these fields as review evidence,
not as adopted policy:

| trace object | source support | why it matters | failure if absent |
|---|---|---|---|
| Public target | `RESEARCH-AGENDA.md`, `TESTS.md`, and RQ5/T3 scaffolds. | Shows the contributor could find a bounded live question. | Review depends on private founder steering. |
| Contributor/accountability context | RQ1 disclosure matrix and RQ5 field-parity check. | Shows identity, pseudonymity, agent assistance, conflict, and source limits are review-relevant without being gatekeeping. | Attribution, contestability, or capture-risk context disappears. |
| Primary class and secondary effects | RQ2/T1 scaffolds. | Prevents mixed work from hiding governance, reward, or semantic movement under a harmless label. | Maintenance, synthesis, implementation, protocol design, and research collapse into private judgment. |
| Evidence minimum | RQ2, RQ3, and T3 scaffolds. | Shows why research, review, formalization, protocol design, or maintenance was evaluated by the right standard. | A reviewer fills gaps with taste or founder context. |
| Value-rationale note | RQ3/T2 scaffolds. | Separates clarity, rigor, relevance, novelty, legitimacy, capture resistance, and coordination value from score totals. | Scores look like authority, reward, or popularity. |
| State-specific rationale | RQ4 scaffold. | Lets a future reader reconstruct the decision and what remains open. | `accepted`, `needs_revision`, `not_accepted`, or `contested` becomes opaque. |
| Protected-boundary sentence | RQ1-RQ5 scaffolds. | Stops claim, governance, reward, rights, appeal, public posture, or reviewer-authority movement before it becomes hidden policy. | Useful work silently changes protected state. |
| Loss notes or residue | T10/T11 and RQ7/RQ9-adjacent scaffolds. | Preserves useful value from narrowed, revised, or non-accepted work. | The repo either erases useful residue or creates reward/rights expectations. |
| Log eligibility decision | T10/T11 and RQ5 scaffolds. | Distinguishes synthetic, draft, reviewed, and live contribution records. | A dry-run or proposal is mistaken for a real contribution log entry. |
| Contestability marker | RQ4/RQ5/T11 scaffolds. | Names what can be disputed without inventing an appeal system. | Challengeability becomes either invisible or overpromised. |

## What Is Ready For Review-Prep Use

These are synthesis conclusions about scaffold readiness, not adoption
decisions:

- The repo has enough first-ring scaffold material to draft a future first-pilot
  review packet without inventing new concepts.
- The safest hypothetical first-pilot shape remains a bounded, source-backed
  research or review contribution, because it can test evidence, rationale, and
  residue without demanding governance or reward movement.
- The main remaining weakness is not missing vocabulary. It is whether a real
  packet can preserve continuity across all fields without a reviewer relying
  on private founder context.
- The current scaffolds repeatedly warn against the same failure: a useful note
  becoming active policy, authority, reward, rights, public posture, or claim
  status because the boundary was not written down.
