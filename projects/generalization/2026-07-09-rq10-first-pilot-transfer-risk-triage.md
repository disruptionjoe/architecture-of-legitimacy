# RQ10 First-Pilot Transfer-Risk Triage

Status: draft transfer-risk triage scaffold, not adopted transfer policy, C8
movement, portability claim, public posture, target selection, governance,
rights, reward, AI policy, or contribution-record policy.

This triage turns the RQ10 first-pilot transfer screen and component inventory
into a review queue. It does not choose a target context, claim that any
component is portable, export governance-package machinery, launch a pilot,
invite contributors, change transfer language, change C8 status, or create
reward, rights, governance, sanction, disclosure, portability,
public-posture, or external-consequence commitments.

## Purpose

The component inventory separates first-pilot pieces into
`portable_candidate`, `adaptable`, `local_only`, and `unknown`. The next risk is
that those labels could be read as a soft transfer plan.

This file asks a narrower question:

```text
What must stay blocked before any RQ10 review treats the first-pilot stack as
transfer evidence?
```

The useful output is a triage layer, not a decision layer. It names why each
possible movement remains blocked and what evidence would be needed before a
later governed review could reopen it.

## Triage Labels

| label | meaning | forbidden inference |
|---|---|---|
| `source_trace_blocked` | Source-repo evidence is still synthetic, incomplete, or not publicly inspectable enough to support transfer review. | Synthetic coherence proves the workflow works. |
| `target_packet_blocked` | A target-specific packet would be needed before the component can be discussed outside this repo. | A context type has been selected or ranked. |
| `second_ring_blocked` | Governance, rights, capture, reward, AI, or transfer-language gates are unresolved. | The component can route around RQ6-RQ9 or C7/T8. |
| `public_language_blocked` | Stronger words would overstate C8. | The repo can say ready, proven, generalizable, or portable. |
| `governed_review_required` | The next move would create target selection, authority, public posture, or external-consequence pressure. | Unattended Progress can make the decision. |
| `local_review_prep_only` | The artifact can help later local review but cannot move transfer state. | The review-prep artifact is transfer evidence. |

These labels are not contribution states, review states, or transfer states.
They only protect later RQ10 review from hidden movement.

## Transfer Review Queue

If later RQ10 work continues, the safest order is:

1. `source_trace_set`: gather or synthesize the minimum accepted, revised,
   rejected, and contested source-repo trace requirements without claiming that
   any live trace exists.
2. `c8_language_boundary`: list words and claims still forbidden while C8 is
   horizon-only.
3. `synthetic_target_packet`: fill one synthetic target-context packet with
   invented values only, without naming a real repo, client, civic setting, or
   funding process.
4. `process_drag_check`: compare where the workflow might add legitimacy and
   where it might add ceremony.
5. `second_ring_gate_read`: inspect which RQ6, RQ7, RQ8, RQ9, or C7/T8 gate
   would fail first in the synthetic target packet.

That queue is a review aid only. It does not approve any of the items.

## Evidence That Still Does Not Count

These are useful local prep, but they should not be counted as transfer
evidence:

- a coherent synthetic first-pilot packet;
- a complete source-trace field list;
- a component label of `portable_candidate`;
- a target-context type named as an example;
- a governance-package portability case study;
- a no-reward boundary in the source repo;
- an AI non-authority trace in the source repo;
- or a public-language boundary that merely says what not to claim.

Each item can lower review confusion. None can establish portability by itself.

## Safe Local Consequence

This triage allows one narrow conclusion:

```text
The first-pilot stack now has a transfer-risk review queue, but every transfer
movement remains blocked before target selection, C8 movement, public posture,
policy adoption, external consequence, or portability claim.
```

That conclusion is intentionally weaker than transfer readiness. It preserves
the value of RQ10 prep while keeping C8 horizon-only.
