# RQ10 C8 Public-Language Boundary

Status: draft public-language boundary scaffold, not adopted public posture,
transfer policy, C8 movement, portability claim, target selection, governance,
rights, reward, AI policy, contribution-record policy, or external action.

This boundary check turns the RQ10 transfer screen, component inventory, and
transfer-risk triage into a wording guardrail for later governed review. It
does not edit `CLAIMS.md`, strengthen C8, choose a target, publish transfer
claims, export governance-package machinery, launch a pilot, invite
contributors, or create reward, rights, governance, sanction, disclosure,
portability, public-posture, or external-consequence commitments.

## Purpose

C8 is currently a horizon claim:

```text
Repo-scale success may generalize to broader collaborative institutions.
```

Recent first-pilot review-prep artifacts make the local stack more coherent,
which creates a predictable overclaim hazard. A coherent synthetic stack can
sound like transfer evidence even while it remains only source-repo review
preparation.

This file asks a narrow question:

```text
Which words and frames must stay blocked until RQ10 has live source traces,
target-context evidence, second-ring clearance, and governed public-language
review?
```

The useful output is a boundary check, not approved copy. It helps later review
notice when C8 language would move faster than evidence.

## Narrow Language That Remains Safe

The following frames are safer for internal review-prep discussion because they
state missing evidence instead of implying movement:

- "C8 remains horizon-only."
- "Transfer evidence is still missing."
- "The current first-pilot stack is source-repo review preparation."
- "The transfer-risk queue names blockers, not approvals."
- "Component labels are review-prep labels, not portability findings."
- "No target context has been selected, ranked, or recommended."
- "Public-language movement requires governed review."

These are not approved marketing, launch, or public-announcement sentences.
They are local boundary phrases for later review.

## Evidence Threshold Before Stronger Language

Before any later RQ10 packet reopens public-language movement, it should show:

1. source-repo live traces for accepted, revised, rejected, and contested paths;
2. a target-specific packet with evidence norms, unsafe evidence, and substitute-evidence rules;
3. a local-only component inventory for the target context;
4. reward and rights deferral language that the target can actually preserve;
5. capture and AI boundary checks specific to the target pressure profile;
6. a process-drag check showing that visible rationale adds more legitimacy than ceremony;
7. abort criteria that would reject transfer before launch;
8. and explicit governed review of any proposed C8, public-posture, or transfer wording.

Until those exist, stronger wording should remain blocked even when local
review-prep artifacts become more coherent.
