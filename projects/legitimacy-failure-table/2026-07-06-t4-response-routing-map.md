# T4 Legitimacy Failure Response Routing Map

Status: synthetic test fixture.

Purpose: pressure-test whether the existing legitimacy failure table can route a diagnosed failure to the right response lane without silently changing policy, claim status, governance rules, reward meaning, or live contribution records.

This file is not a legitimacy-condition change, claim-status decision, governance change, contribution policy, appeal rule, reward rule, rights policy, public-posture change, or evidence that T4 has passed. It is a bounded routing map for later review.

## Routing Lanes

| lane | use when | safe local output | stop boundary |
|---|---|---|---|
| Local workflow repair | The failure can be made more visible, reviewable, or traceable with fields, links, rationale, states, or synthesis. | Draft fixture, checklist, template note, review prompt, or non-adopted test artifact. | Stop before changing live policy, reviewer powers, rights, rewards, claims, or public posture. |
| Evidence gathering | The failure is plausible but the repo lacks enough examples or signals to distinguish design flaw, execution flaw, dissatisfaction, or attack. | Synthetic case, evidence checklist, metric proposal, or future review question. | Stop before treating a metric as an authoritative judgment rule. |
| Joe review | The response would adopt or alter contribution policy, governance authority, participant rights, disclosure rules, appeal meaning, sanctions, public commitments, or claim status. | Clearly framed review packet or question. | Do not implement the policy move without Joe's explicit approval. |
| Governance/capture stop | The failure suggests rule-change capture, founder exception drift, reward pressure, concentrated reviewer authority, or adversarial manipulation. | Boundary map, capture-risk note, rollback question, or escalation packet. | Do not normalize the contested rule, exception, reward path, or power shift as a local cleanup. |

The useful distinction is not "small versus large" work. The useful distinction is whether the response improves inspectability inside the current rules or changes the rules participants rely on.

## Boundary Notes

- A local workflow repair can make future policy decisions easier, but it must not quietly become policy.
- Evidence gathering should expose uncertainty rather than convert early metrics into authority.
- Joe review is required when the response changes rights, obligations, governance powers, public promises, claim status, or reward meaning.
- Governance/capture stops should preserve the contested condition long enough for review rather than smoothing it into a normal process update.
- A failed contribution review can still be legitimate when rationale, contest path, retained record, and rule-learning path remain visible.
