## Why This Shell Is Needed

The evidence boundary says later portability review needs brownfield-conflict
evidence, not just origin-function or transfer-behavior evidence. A component
can solve a real origin problem and survive transfer while still duplicating,
displacing, confusing, or weakening native surfaces in a mature target
repository.

The origin-function and transfer-behavior shells make source and transfer rows
fillable. This shell makes brownfield-conflict rows fillable without crossing
the source-use or target-context gate. It keeps native-surface overlap,
duplicate authority, process drag, and target-fit claims blank until a governed
pass can apply publication review and cite only public-safe evidence.

## Packet Use Rule

Use this artifact only as a blank packet scaffold. A later run may fill a row
only after it has applied the source-evidence publication gate and can record
the source visibility label, public-safe target-context fact, withheld-detail
class, native overlap or conflict surface, reviewer inference, evidence
necessity, reusable-process risk, and stop boundary.

Unknown source visibility defaults to `private_or_internal_source`. Reusable
governance/process detail and target-specific process detail stop for Joe or
governed review before public write-up.

## Review Sequence

1. Apply the source-evidence publication gate.
2. Fill only public-safe facts, or write `none` when the detail is blocked.
3. Separate target-context fact, native-surface observation, reviewer inference,
   and hypothesis.
4. Name whether the component duplicates, conflicts with, complements, or
   leaves untouched the native surface.
5. Apply the process-drag check before any tentative value judgment.
6. Apply the do-not-port signal register before any portability discussion.
7. Leave portability class blank until evidence exists and governed review
   authorizes classification.
