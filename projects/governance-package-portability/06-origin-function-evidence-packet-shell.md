## Why This Shell Is Needed

The inventory frame says the next legitimate output is an evidence packet, not
an adoption plan. The evidence boundary narrows the safest material study to
an origin-function packet for two or three components. The publication gate
then blocks a public evidence packet until source visibility, reusable-process
risk, and protected-consequence screens are applied.

This shell makes the next packet concrete without crossing the source-use gate.
It gives a later governed source pass a narrow table to fill and keeps every
blank row from being mistaken for a finding.

## Packet Use Rule

Use this artifact only as a blank packet scaffold. A later run may fill a row
only after it has applied the source-evidence publication gate and can record
the source visibility label, public-safe fact, withheld-detail class, reviewer
inference, evidence necessity, reusable-process risk, and stop boundary.

Unknown source visibility defaults to `private_or_internal_source`. Reusable
governance/process detail stops for Joe review before public write-up.

## Minimum Later Row Shape

When a later governed pass fills one of the rows, keep these fields together.

| field | required content before review |
|---|---|
| Component | One of the candidate governance-package components named in this shell, or a separately justified addition. |
| Institutional problem | The local problem the component is supposed to solve. |
| Source visibility label | One label from the source-evidence publication gate. |
| Public-safe source fact | The narrow fact that can be written in this public repo, or `none`. |
| Withheld detail | The class of withheld detail without disclosing the detail itself. |
| Origin observed effect | What actual work changed because the component existed, if publicly writable. |
| Reviewer inference | Cautious interpretation separated from source facts and observed effects. |
| Evidence necessity | Why this detail is needed for the portability question. |
| Process-drag signal | Whether the component reduced decision cost or added ceremony, staleness, duplicate authority, or attention tax. |
| Do-not-port signal | Any deferral, rejection, thinning, or removal signal from the signal register. |
| Kill condition | Evidence that should stop, defer, or remove the component from portability consideration. |
| Stop boundary | Joe-review, governed-review, public-posture, rights, reward, governance, or cross-repo action boundary. |

## Review Sequence

1. Apply the source-evidence publication gate.
2. Fill only public-safe facts, or write `none` when the detail is blocked.
3. Separate source fact, observed effect, reviewer inference, and hypothesis.
4. Apply the process-drag check before any tentative value judgment.
5. Apply the do-not-port signal register before any portability discussion.
6. Leave portability class blank until evidence exists and governed review
   authorizes classification.
