## Why This Shell Is Needed

The evidence boundary says later portability review needs transfer-behavior
evidence, not just origin-function evidence. A component can solve a real
problem in its origin setting and still become stale, too heavy, duplicated,
or authority-confusing after transfer.

The origin-function shell makes origin rows fillable. This shell makes the
transfer-behavior rows fillable without crossing the source-use gate. It keeps
changed, broken, stable, and stale-transfer claims blank until a governed pass
can apply publication review and cite only public-safe evidence.

## Packet Use Rule

Use this artifact only as a blank packet scaffold. A later run may fill a row
only after it has applied the source-evidence publication gate and can record
the source visibility label, public-safe fact, withheld-detail class, transfer
observed effect, reviewer inference, evidence necessity, reusable-process risk,
and stop boundary.

Unknown source visibility defaults to `private_or_internal_source`. Reusable
governance/process detail stops for Joe review before public write-up.

## Review Sequence

1. Apply the source-evidence publication gate.
2. Fill only public-safe facts, or write `none` when the detail is blocked.
3. Separate source fact, transfer observed effect, reviewer inference, and
   hypothesis.
4. Name whether the component changed, stayed useful, broke, or became stale.
5. Apply the process-drag check before any tentative value judgment.
6. Apply the do-not-port signal register before any portability discussion.
7. Leave portability class blank until evidence exists and governed review
   authorizes classification.
