## Why This Trace Is Needed

The academic peer-review source packet is strong enough to name a meaningful
comparison surface: expert validation, editorial authority, revision,
publication record, appeals, post-publication critique, transparent-review
variants, and opacity limits.

The unresolved gap is adverse-path visibility. Accepted publications may leave a
public record, especially when a journal publishes peer-review history. Rejected,
desk-rejected, filtered, withdrawn, or heavily revised work often leaves a much
weaker third-party trace. That matters for T9 because this repo's local burden is
not only whether a system validates knowledge. It is whether a future reviewer
can reconstruct contribution evidence, rationale, loss notes, retained record,
retained recognition, and any explicitly bounded future-value effect.

## Candidate Path Types

A later T9 review should choose exactly one path type before filling the trace.

| path type | why it pressures the local claim | main evidence risk |
|---|---|---|
| Desk-rejected manuscript | Tests whether editorial filtering leaves enough rationale or record for later reconstruction. | The rejection path may be private, terse, or inaccessible. |
| Rejected after external review | Tests whether reviewer comments, editor rationale, and appeal options preserve visible loss notes. | Review files may be confidential unless authors or venues publish them. |
| Major-revision path that becomes accepted | Tests whether revisions, response letters, and published review history preserve contribution narrowing rather than only final success. | Accepted-publication traces may hide rejected alternatives and compressed contribution history. |

The later reviewer should trace one actual peer-review path, not an abstract

## What Would Count As Trace-Ready

A later filled trace is trace-ready only if it can separate:

- directly sourced process facts;
- reviewer inference;
- what the contributor, author, reviewer, editor, or public can inspect;
- what survives after rejection, revision, acceptance, or review labor;
- whether the surviving record does any later work beyond recognition;
- what is explicitly not promised; and
- which local protocol surface is pressured.

Label-level references to "peer review has appeals" or "peer review has
publication records" are not enough. The trace needs a concrete path showing
what happens to the adverse or narrowed work.

## Non-Promissory Comparison Language

The local comparison should remain negative-boundary language until Joe or a
later governance path authorizes otherwise:

```text
This peer-review path may preserve record or recognition, but it does not by
itself establish a payout, ownership claim, governance weight, guaranteed review
priority, claim-status change, or retained-value marker for this repo.
```

If a later trace cannot say that cleanly, it has crossed into rights, reward,
governance, legal, financial, or public-posture territory and should stop for
review.
