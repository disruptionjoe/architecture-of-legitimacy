## Purpose

The synthetic first-pilot packet currently treats a bounded source-backed
prior-art or mechanism-comparison note as the safest possible first-pilot
shape. That shape is useful because it stays close to research. It is also
risky because it can accidentally imply that T9 prior-art review is mature
enough to carry a novelty conclusion.

This screen asks a narrower question:

```text
What prior-art checks would a later first-pilot packet need before a bounded
source-backed research/review contribution could be reviewed without implying a
T9 verdict?
```

The answer here is review-prep only: the packet can be shaped, but the
prior-art screen remains incomplete until a concrete source lane and source
trace are selected through a governed path.

## Minimum Prior-Art Fields For A Later First-Pilot Packet

Any live or live-like packet should fill these fields before review:

| field | question |
|---|---|
| Targeted local surface | Which test, research question, claim-adjacent scaffold, or project artifact is the note trying to clarify? |
| Neighboring system | Which one prior-art lane or neighboring system is under comparison? |
| Contribution unit | What exact unit is being compared: application, edit, PR, issue, manuscript, review labor, governance action, funding round, or case record? |
| Source facts | Which facts are directly supported by the cited source material? |
| Reviewer inference | Which comparisons are interpretation, not source fact? |
| Adverse-path status | Is this accepted-path-only, adverse-path, under-valued-path, or mixed-path evidence? |
| Mechanism axes | Which one or two mechanisms are compared: validation, allocation, voice, contestability, record, reward, governance, capture, or coupling? |
| Similarity pressure | What looks similar enough to reduce overclaim? |
| Difference pressure | What remains different enough to require more review? |
| Missing evidence | What source or trace would be needed before collision level review? |
| Protected boundary | What the packet does not decide about claims, tests, policy, governance, rights, rewards, public posture, transfer, or live records. |

If those fields cannot be filled, the packet should remain `needs_revision` or
`source_trace_missing`, not reviewed contribution material.
