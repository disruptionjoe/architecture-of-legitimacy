# Roadmap

The roadmap is organized by research maturity, not hype.

## Phase 0: Repo Launch

Goal:

Create a public research base that can receive useful contributions.

Deliverables:

- essay,
- claim ledger,
- protocol stack,
- legitimacy schema,
- threat model,
- tests,
- contribution standards,
- contribution log,
- governance phase model.

Status:

Initial launch.

## Phase 1: First Contribution Workflow

Goal:

Make one valid external contribution possible without private context.

Deliverables:

- issue templates,
- contribution proposal template,
- first review cadence,
- contribution log schema,
- three sample contribution entries,
- first accepted/revised/rejected examples.

## Phase 2: Formalization

Goal:

Turn core claims into models and failure conditions.

Deliverables:

- collaboration-versus-defection toy model,
- attack-profitability model,
- legitimacy failure taxonomy,
- contribution value rubric tests,
- prior-art collision review.

## Phase 3: Pilot Review

Goal:

Run real contributions through the process.

Deliverables:

- public review notes,
- contested decision workflow,
- monthly synthesis,
- rubric revision history,
- participant feedback summary.

## Phase 4: Governance Transition Design

Goal:

Make progressive decentralization observable rather than theatrical.

Deliverables:

- named founder powers,
- transition milestones,
- first transferable powers,
- reviewer selection criteria,
- appeal governance.

## Phase 5: Generalization

Goal:

Decide whether the pattern can move beyond one repo.

Deliverables:

- transfer criteria,
- candidate second repo,
- public-goods use-case comparison,
- institutional design paper or long-form essay.
