## Failure / No-Go Mining

- Recent wall: the previous dry run showed the workflow is usable, but it used a synthetic contribution with omitted citations. The next wall is not template mechanics; it is whether real source-backed review can avoid hidden founder context.
- Negative result worth preserving: if the first real contribution lands as `needs_revision` or `not_accepted`, that is valuable evidence if the rationale and loss notes are visible enough for a future contributor to reconstruct the decision.
- Assumption under pressure: adding `loss_notes` preserves legitimacy only if reviewers actually use the field to record filtered, compressed, or deferred value without implying a reward promise.
- Assumption under pressure: "open attemptability" may still depend on private taste if the repo has not named minimum quality and evidence thresholds for the first pilot.
- Assumption under pressure: the governance-package portability study belongs here only while it functions as a legitimacy case study; it becomes risky if it starts importing or exporting reusable process before evidence.
- What would change the current priority: a real external contribution, a serious prior-art collision showing the coupled-stack claim is less novel than expected, or evidence that the intake workflow creates too much noise or ambiguity.

## Hidden Assumptions And Better Questions

1. Hidden assumption: first contribution success means acceptance. Better question: can the repo make an adverse or revision decision feel legitimate because the evidence, rule, rationale, contestability, and loss trace are visible?
2. Hidden assumption: the contribution log can later support rewards if it records enough fields. Better question: which fields preserve provenance and loss without creating implied legal, financial, or governance rights?
3. Hidden assumption: founder-led review is acceptable if documented. Better question: which founder judgments are taste/coherence judgments, which are governance judgments, and which require explicit appeal or monthly synthesis?
4. Hidden assumption: prior-art comparison is one research task among many. Better question: which prior-art collision would most quickly weaken or sharpen C2/C3 without diffusing attention into a survey project?
5. Hidden assumption: portability work is an extension of this repo's legitimacy thesis. Better question: when does studying governance portability become reusable IP extraction that should route to CapacityOS or another governed surface?
