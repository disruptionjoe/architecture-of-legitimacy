## North Star

This repository has one North Star with two components. The **blockbuster
component** is the claim Lane 1 forces or falsifies. The **agent-capability
component** is not a lane, never justifies an experiment on its own, and does two
other jobs: it explains why this repository should chase this particular claim,
and it constrains how every Lane 1 experiment is run.

### Blockbuster component

Develop and force-or-falsify:

> There are **findable conditions** under which enduring legitimacy is
> constructible in a positive-sum game — conditions that make corruption and
> collusion **nonviable rather than merely punished**, and that hold because of
> what the enforcement substrate makes possible rather than because of what any
> population currently prefers.

This is the highest-ceiling target, not a premise.

The load-bearing words are *nonviable* and *findable*. A system that detects and
punishes corruption after the fact is an enforcement regime; the claim is that
there exist conditions under which the corrupt move is not available to make. And
*findable* means the conditions are discoverable by method rather than asserted by
a designer — if they can only be stipulated, there is no science here, only
preference with a governance file attached.

If it holds, institutional design acquires a feasibility boundary: for a given
enforcement substrate, some legitimacy conditions are constructible and others are
impossible regardless of intent, and the difference is computable in advance rather
than discovered by failure.

### Agent-capability component

> This repository tests whether research agents can perform **adversarial
> self-application**: run the instrument on the institution that is running the
> instrument, and return a verdict its host did not want — without flattering the
> host, and without reaching for the host's own ratified escape hatch.

**Why this repository chases this blockbuster.** No other repository in the
portfolio is its own specimen. GU Formalization, Time as Finality, Temporal
Issuance, Possibility to Capability, and Continuity Ledger all measure something
external; their neutrality gates guard against *preferring an outcome*. This
repository's hazard is structurally different: **preferring a self.** That cannot
be fixed by blinding, because an agent cannot be blinded to which repository it is
running in. And this repository is a live, self-owned, fully instrumented instance
of the object it studies — a recognition surface with irrevocable conferral, a
founder holding undelegated powers, and every frame-setting decision timestamped in
git. For any other repository that would be fatal contamination. Here it is the only
specimen anyone has where the frame-setter's choices are on the record.

**How it constrains method.** See `## Self-Application Discipline`.

This is a research and evaluation role, not evidence that agents already have the
capability. Present evidence is against it: `AGENTS.md` carries a standing warning
against formalizing a finishable byproduct instead of pursuing the North Star, and
`projects/` then acquired a read-order for its own maps and a coverage audit of its
own index. Recognizing a failure mode in one's own charter did not prevent enacting
it one directory over. Reading a warning is not obeying it.

### The components are separate

Neither component is evidence for the other. A successful adverse self-verdict is
never evidence that the legitimacy conditions exist. A finding about the
conditions is never evidence that the agent system holds the capability.

The agent-capability component may not commission an experiment. It may only
explain the choice of target and constrain method. An experiment proposed because
it would test the agents is out of scope regardless of its quality.

## Self-Application Discipline

The method constraints the agent-capability component places on Lane 1 work. A run
that violates one is inadmissible regardless of its result.

1. **Measure the host first.** Where a claim is testable on this repository's own
   substrate, it is tested here before it is tested anywhere else. The first
   experiment is on git (see `## First Experiment`).
2. **An adverse finding about this repository is a first-class result**, published
   at the same grade as a favourable one, including the finding — if it is the
   finding — that the founder-led phase places this repository outside its own
   feasible interval.
3. **No rule-change under adverse measurement.** See `P4` below. Amending the
   instrument while it is returning an unwelcome number is the failure mode this
   repository exists to detect. An agent that does it here has disqualified the run.
4. **Self-criticism is not a credibility move.** Producing an adverse finding in
   order to appear rigorous is the same defect as suppressing one. Both substitute
   the appearance of a verdict for a verdict.
5. **Name the founder's discretion wherever it bears.** Every measurement on this
   repository states which of its inputs the founder controls. An unnamed discretion
   is an unrecorded import.

## Ratified Prohibitions

These are **prohibitions, not absences.**

Before this charter, this repository could not cross the seam with Continuity
Ledger because it *lacked the apparatus* — no typing discipline, no `M0`/`M1`, no
absorber gate, no covariance requirement. **An absence is not a prohibition.** An
absence is repaired by the next capable contributor, silently and in good faith,
and the seam dissolves without anyone deciding to dissolve it. Continuity Ledger's
half is already structural: its charter states `legitimacy is not information`, so
a legitimacy verdict fires its typing gate, and its neutrality gate fires
independently. This repository now ratifies its half.

### P1. No substrate-quantity issuance.

This repository does not measure, estimate, model, or issue exit costs, chainwork,
issuance schedules, thermodynamic quantities, or any substrate-physical quantity.
It **consumes** such quantities from a frozen, provenance-bearing Continuity Ledger
packet and issues a legitimacy verdict upon them. If no such packet exists, the
correct result is `NOT_YET_ADMISSIBLE`, naming the missing source gate. An agent
that computes a substrate quantity here has voided the seam, and the result is
inadmissible regardless of its quality.

**Exception, narrow and deliberate:** this repository's own substrate. `P1` binds
verdicts about *other* systems. Measuring the git substrate this repository runs on
is self-application, not seam-crossing, and is required by `## First Experiment`.
Without this exception `P1` would strip this repository of every instrument that
can kill it, and a repository whose sole falsifier is imported has a kill condition
in name only.

### P2. Legitimacy is not a preference aggregate.

A legitimacy verdict issued here may not be a function of measured, reported, or
inferred participant preference. Preference selects a position within the feasible
conditions; it does not set which conditions are feasible. **A verdict that would
change if the bound population's preferences changed, at fixed substrate and fixed
frame, is not a verdict of this repository — it is a survey**, and it falsifies this
repository's North Star rather than instantiating it.

### P3. No frame authorship.

This repository does not declare Φ, `R0`, or `S0`. Those are declared upstream and
imported. A verdict computed at a frame this repository chose is inadmissible: the
blockbuster's independent variable is the substrate, and a repository that both
selects the substrate and evaluates its consequences has proved a tautology with a
dial on it.

### P4. Non-amendability under adverse measurement.

The sections `## North Star`, `## Ratified Prohibitions`, `## Kill Condition`, and
`## Self-Application Discipline` are **outside the scope of Legitimacy Condition 5
(Adaptive Rule Change)**. Condition 5 governs rubrics, cadences, taxonomies,
workflows, and the schema. It does not govern this file.

An amendment to these four sections proposed while a run adverse to them is open is
inadmissible until that run closes and publishes.

Rule-change-as-escape is the specific failure mode this repository exists to detect
in others. A charter that pre-authorizes it cannot detect it in anyone.

## Kill Condition

This repository had no kill condition. It has one now, and it is written before any
conclusion exists that would make it convenient not to have one.

**This repository is killed if either holds:**

**K1 — the git prediction fails.** On the git substrate, capture cost is *not* a
non-decreasing function of accumulated legitimacy and participation — or its
observed resistance is traceable to founder decisions rather than to substrate
properties. See `## First Experiment`. If capture resistance comes from the founder
rather than from git, then legitimacy conditions are not substrate-determined here,
and the central clause is false on the one substrate this repository can measure
directly.

**K2 — the conditions track the evaluator.** The feasible legitimacy conditions
vary with the evaluator index at fixed substrate and fixed frame. That is the
preference account — the thing this repository was founded to contradict — and its
confirmation ends this repository as a research program.

A third outcome kills it as vacuous rather than wrong: if the conditions are
invariant under change of substrate as well, then the substrate is not a variable
and there is nothing here to find.

**Prohibited responses to a firing kill condition**, each named because each is
available and each is what a degenerating programme reaches for:

- re-declaring the frame;
- adding an evaluator-index covariate;
- narrowing to "legitimacy is multiple" — this is preference-relativism with a
  citation, it is the maximally defensible response, and it teaches nothing;
- adding governance mechanism until the number improves, which converts a
  prediction back into a goal and voids the test; and
- invoking Condition 5.

Adaptive rule change is a legitimacy condition of the institutions this repository
studies. **It is not a method of this repository's own science.**

## First Experiment

**The git Non-Capture measurement.** Selected by the agent-capability component:
run the instrument on the host.

`LEGITIMACY-SCHEMA.md` Condition 7 currently reads as a design goal — "the system
should become harder to capture as legitimacy and participation grow." As a goal it
is unfalsifiable. Restated as a prediction about this repository's own substrate:

> On the git substrate, capture cost is a non-decreasing function of accumulated
> legitimacy and participation — **because of properties of git** (fork
> availability, replicated history, unforgeable authorship) **and not because of
> anything this project decides.**

This is measurable today, on this repository, with no packet from any other
repository. It can kill this repository unilaterally (`K1`). And it is the one
experiment where the blockbuster and the capability are the same act: measuring
whether *our* legitimacy raises *our* capture cost, and reporting it if it does not.

Stated consequence, in advance: if capture cost is flat or falling in accumulated
value on git, then legitimacy does not raise capture cost on this substrate, this
repository's proving ground is the wrong one, and the honest response is to name the
substrate on which it might hold — **not** to add a governance mechanism until the
number improves.

## Second Experiment: the Interval Sweep

The seam experiment with Continuity Ledger. This repository owns the legitimacy half
of the verdict and consumes the substrate half as a frozen packet (`P1`). It may not
read the exit-cost half's construction before its own arm is issued.

This experiment does **not** carry this repository's falsifiability — `K1` does. That
separation is deliberate: an earlier design gave this repository a kill condition that
could only fire on a finding produced by another repository, which is a kill condition
in name only.

## Success Standard

The program succeeds if it produces at least one of:

- a demonstrated condition under which corruption or collusion is nonviable rather
  than punished, on a named substrate, with its feasibility boundary stated;
- a proof that no such condition exists on a named substrate — a no-go for
  institutional design on that substrate;
- a measured capture-cost curve for a real substrate, including this repository's
  own, that discriminates substrate-determined resistance from founder-determined
  resistance; or
- a reproducible finding about whether research agents can return an adverse verdict
  about their own host without reaching for its ratified escape hatch.

A clean kill is a successful result.
