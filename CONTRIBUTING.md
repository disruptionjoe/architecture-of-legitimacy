# Contributing

This project is in its founding research phase. The most useful contributions are precise, bounded, and truth-seeking.

## Before You Contribute

Read:

- [README.md](README.md),
- [CLAIMS.md](CLAIMS.md),
- [RESEARCH-AGENDA.md](RESEARCH-AGENDA.md),
- [CONTRIBUTION-STANDARDS.md](CONTRIBUTION-STANDARDS.md),
- and [GUARDRAILS.md](GUARDRAILS.md).

## Good First Contributions

Strong first contributions include:

- improving one claim in the claim ledger,
- adding prior art with a short relevance note,
- proposing a better value rubric,
- red-teaming one protocol component,
- refining the toy payoff model,
- drafting a contribution-log schema,
- or identifying an overclaim.

## Contribution Proposal Format

Use this structure:

```md
## Contribution Type

research | formalization | protocol design | review | synthesis | implementation | maintenance

## Target

Which claim, test, document, or research question does this address?

## Contribution

What are you adding or changing?

## Why It Matters

How does this improve clarity, rigor, legitimacy, capture resistance, or contributor usability?

## Evidence Or Reasoning

What sources, examples, models, or arguments support it?

## Failure Or Risk

What might be wrong with this contribution?

## Loss Or Deferred Value

What useful value might be lost if this contribution is revised, narrowed, or not accepted as-is?
```

## Review Expectations

The project may classify contributions as:

- `accepted`,
- `needs_revision`,
- `not_accepted`,
- `contested`,
- or `adversarial`.

Early review is founder-led. That should change only when review standards become clear enough to distribute responsibly.

## AI-Generated Contributions

AI-assisted contributions are allowed when labeled.

They should include:

- what tool or process was used,
- what human review occurred,
- and which claims or documents were affected.

AI-generated volume without human judgment may be rejected as noise.

## Code Of Conduct

Be rigorous and direct without being abusive.

This project values critique. It does not value harassment, manipulation, plagiarism, or bad-faith flooding.
